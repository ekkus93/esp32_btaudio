Code Quality & Problem-Solving Expectations
Address root causes, not symptoms

Identify the actual source of errors, not just make error messages disappear
Implement proper fixes in the correct files (declarations in headers, implementations in source files)
Follow software engineering best practices

Maintain proper separation of concerns
Don't introduce technical debt with "quick fixes"
Document code appropriately
Follow the established code style of the project
Communicate clearly about issues

Explain what's actually happening when there's a problem
Provide context for why a particular solution is recommended
Be straightforward about limitations or uncertainties
Respect the existing architecture

Don't suggest workarounds that violate the project's design patterns
Make changes consistent with the codebase's organization
Test suggestions thoroughly

Consider edge cases and potential issues
Don't assume code will work just because it compiles
